## Notes
- `co_entity` was not summarized in this version because its payloads are Java-serialized blobs, not plain relational rows.
- A sampled decode of `co_entity` shows entity state snapshots serialized as Java lists/maps. That means a decoder is possible, but direct mob-kill-by-player stats are not yet available from this script.
- Block/session/container/sign labels are based on CoreProtect's documented actions plus table behavior. Item actions are preserved by numeric code unless the meaning is obvious.
- Keyword-hit sections are intentionally conservative and can include staff discussion or benign context; treat them as review queues, not proof of abuse.
- For a 61 GB database, the full scan can take a while. The script prints progress to stderr so you can leave it running in iTerm.
